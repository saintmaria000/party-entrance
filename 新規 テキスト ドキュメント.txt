# ? Party Entrance WEB
## 要件定義

---

## 1. 目的

本サイトは、野外パーティーの詳細情報を一般公開せず、限定的に案内するためのWebサイトである。

ユーザーは URL または QRコード から本サイトにアクセスし、名前およびメールアドレスを入力することで、イベント情報を受け取ることができる。

本サイト上には必要最小限の情報のみを表示し、詳細情報はフォーム送信後の自動返信メールにて案内する。

---

## 2. サイトの役割

・参加希望者の導線をつくる  
・参加希望者の名前とメールアドレスを取得する  
・自動返信メールで必要情報を送る  
・送信完了後にメッセージページを表示する  
・メッセージページでミックス音源のダウンロード導線を提供する  

---

## 3. 情報開示設計

### Level1：フォームページ
・最低限のコピー表示  
・Name / Mail 入力  
・送信導線  

### Level2：メッセージページ
・送信完了後に表示する  
・参加者向けメッセージを表示する  
・ミックス音源ダウンロード導線を表示する  

### Level3：自動返信メール
・開催日時  
・開催場所  
・フライヤーダウンロードURL  

---

## 4. 管理対象

・フォーム送信者名  
・フォーム送信者メールアドレス  
・送信日時  

---

## 5. 技術方針

・フロントは HTML / CSS / JS で構成する  
・送信データは Google Sheets に保存する  
・自動返信メールは Resend を利用する  
・送信完了後は message.html に遷移する  
・スマホ表示に対応する  

---

## 6. 外部サービス方針

・送信者管理：Google Sheets  
・自動返信メール：Resend  
・フライヤー配布：URLダウンロード方式  

---

## 7. リスク

・情報不足による不信  
・URL拡散  
・フォーム不具合  
・表示崩れ  
・メール未達  
・スパム送信  

---

## 8. 本質

このサイトは情報を並べるものではなく、必要な人にだけ必要な情報を渡すための導線を設計するものである。

# ? Party Entrance WEB
## 機能要件

---

## 1. ページ構成機能

・form.html を表示できること  
・message.html を表示できること  
・style.css にて共通スタイルを管理できること  
・app.js にてフロント制御を行えること  

---

## 2. フォーム入力機能

・ユーザーが Name を入力できること  
・ユーザーが Mail を入力できること  
・Name / Mail は必須入力であること  
・Mail はメールアドレス形式であること  
・バリデーション通過時のみ送信できること  
・送信中は多重送信を防止できること  

---

## 3. 送信処理機能

・フォーム送信時に入力データを所定の送信先へ送信できること  
・送信成功 / 送信失敗を判定できること  
・送信成功時は即時 message.html に遷移すること  
・送信失敗時は form.html 上で error 表示を行うこと  
・送信失敗後も再送可能であること  

---

## 4. アナウンス表示機能

・画面上部にアナウンスを表示できること  
・状態名は success / error で管理すること  
・success 時は form.html 上で表示せず即時遷移すること  
・error 時のみ form.html 上で表示すること  
・error アナウンスは一定時間後に自動で消えること  

---

## 5. 送信者管理機能

・フォーム送信者の name を保存できること  
・フォーム送信者の email を保存できること  
・送信日時を保存できること  
・保存先は Google Sheets とすること  
・管理者が一覧で確認できること  

---

## 6. 自動返信メール機能

・フォーム送信成功後に自動返信メールを送信すること  
・送信先は入力されたメールアドレスとすること  
・送信には Resend を用いること  
・送信元アドレスを設定できること  

---

## 7. 自動返信メール内容機能

・件名を設定できること  
・本文に以下を掲載できること  
  - 開催日時
  - 開催場所
  - フライヤーダウンロードURL

・本文はテキストベースで成立すること  

---

## 8. メッセージページ機能

・送信完了後に message.html を表示できること  
・メッセージ本文を表示できること  
・ミックス音源のダウンロード導線を設置できること  
・ミックス音源をダウンロードできること  

---

## 9. 更新・変更機能

・開催日時を更新できること  
・開催場所を更新できること  
・フライヤーダウンロードURLを更新できること  
・メール件名を更新できること  
・メール本文を更新できること  
・メッセージページ上の本文を更新できること  
・ミックス音源URLまたはファイルを更新できること  
・フォントファミリーを変更できること  
・余白、フォントサイズ、レイアウト幅を変更できること  

---

## 10. 外部サービス連携機能

・Google Sheets に送信データを保存できること  
・Resend API を利用できること  
・独自ドメインの送信元設定に対応できること  
・フライヤーURLを差し替え可能であること

## デザイン要件（統合版）

---

## 1. 全体方針

・黒背景をベースとする  
・中央配置を基本とする  
・ミニマル構成とする  
・情報を載せすぎない  
・派手な装飾を避ける  
・静かな緊張感を保つ  

---

## 2. レイアウト方針

・背景 / オーバーレイ / コンテンツ の3層構造を持てること  
・form.html は中央配置とすること  
・message.html は縦スクロール前提とすること  
・余計な gap / margin / padding を初期状態で持ちすぎないこと  
・後から余白やサイズを調整しやすい見た目構造にすること  
・背景は原則黒統一としつつ、将来的に画像や映像を差し込める構造とすること  

---

## 3. CSS設計方針

・余白、フォントサイズ、幅は CSS変数で管理すること  
・変数は :root で定義すること  
・.form-page / .message-page 単位で上書きできること  
・固定値の直書きを避けること  
・初期状態は最小構造とすること  

---

## 4. フォント要件

・フォントサイズは CSS変数で管理すること  
・フォントファミリーは CSS変数で管理すること  
・初期フォントは最もシンプルなシステムフォントとすること  

---

## 5. フォームページ表示要件

・body には `form-page` を付与すること  
・ページ全体の基準要素として `.page` を用いること  
・背景拡張用レイヤーとして `.bg-layer` と `.overlay-layer` を配置できること  
・コピーを中央付近に配置すること  
・フォームは `.form` としてまとめること  
・各入力ブロックは `.field` で管理すること  
・入力欄は下線のみの最小限の見た目とすること  
・ボタンはテキスト中心の見た目とすること  
・過度な装飾を加えないこと  
・ページ上部に全体エラー表示領域を持つこと

## 7. アナウンス表示要件

・アナウンスは画面上部に表示すること

## 8. レスポンシブ表示要件

・スマホ表示に対応すること  
・PC表示でも破綻しないこと  
・サイズ調整は CSS変数ベースで行えること  

1) フロントを用意（そのまま）
   form.html / message.html / style.css / script.js

2) Cloudflare Pages にデプロイ
   - GitHub に push
   - Pages プロジェクト作成 → デプロイ

3) API ファイルを作成
   path: /functions/api/entry.js
   → 先ほどの「完全版コード」をそのまま貼る

4) Turnstile を作成（Cloudflare）
   - Security → Turnstile → Add site
   - SITE_KEY と SECRET_KEY を取得
   0x4AAAAAACvvfHHYstTRtB2X

   0x4AAAAAACvvfL8bNOWfg241aIWvqddZaa8

5) HTML に SITE_KEY を設定
   form.html 内：
   data-sitekey="YOUR_TURNSTILE_SITE_KEY"

6) 環境変数を設定（Cloudflare Pages）
   Settings → Functions → Environment Variables
   追加：
   RESEND_API_KEY=（ResendのAPIキー）
   TURNSTILE_SECRET_KEY=（TurnstileのSecret）

7) KV を作成（リスト保存用）
   Workers & Pages → KV → Create namespace
   name: ENTRIES

8) KV をバインド
   Settings → Functions → KV bindings
   variable: ENTRIES
   namespace: ENTRIES

9) API の送信先メールを設定
   /functions/api/entry.js 内：
   const ADMIN_TO = "your-mail@example.com";

10) デプロイ更新
   - 変更を push
   - Cloudflare が自動再デプロイ

11) 動作確認
   - フォーム送信
   - /api/entry が呼ばれる
   - メール届く
   - 重複登録弾かれる
   - message.html に遷移
